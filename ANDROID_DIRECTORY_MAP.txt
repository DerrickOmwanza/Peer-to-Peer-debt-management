              └── ApiClient (HTTP client)

        └── RiskScoreRepository

================================================================================
ARCHITECTURE PATTERN
================================================================================

MVVM (Model-View-ViewModel):

View (Activity) 
    ↓ observes
ViewModel (manages state)
    ↓ calls
Repository (wraps API)
    ↓ calls
ApiService (HTTP)
    ↓
Backend Server

Benefits:
  • Clean separation of concerns
  • Easy to test
  • Lifecycle-aware
  • Survives configuration changes
